# /step-by-step - ステップバイステップ学習スキル

課題やタスクを小さなステップに分解し、1つずつ順番に進めていく。
受講生が迷子にならないように、今どこにいるかを常に示す。

**進捗はMDファイルとして保存し、チェックボックスで管理する。**

## 使い方
```
/step-by-step <やりたいこと or 課題>
```

## 処理内容

### 1. ゴールの確認
- 受講生がやりたいことを確認する
- 最終的にどうなればOKかを明確にする
- 例: 「じゃんけんゲームを作りたい → ターミナルで動く、グーチョキパーを選べる、勝敗が出る」

### 2. ステップの分解 + タスクリストMDファイルの作成
- ゴールを5-8個の小さなステップに分ける
- **同時に進捗管理用のMDファイルを作成する**
- ファイル名: `progress_<課題名>.md`（例: `progress_じゃんけんゲーム.md`）
- ファイルの内容フォーマット:

```markdown
# 📋 <課題名> - 進捗タスクリスト

**ゴール**: <最終的にどうなればOKか>
**開始日**: YYYY-MM-DD

---

## 進捗: 0 / X ステップ完了

- [ ] **ステップ1**: 〇〇の説明
- [ ] **ステップ2**: △△の説明
- [ ] **ステップ3**: □□の説明
- [ ] **ステップ4**: ▽▽の説明
- [ ] **ステップ5**: ◇◇の説明

---

## 学習メモ
（ステップ完了時に学んだことを追記していく）
```

- ファイル作成後、受講生に**ファイルを作成したこと**を伝える
- チャット上にもステップ一覧を表示する

### 3. 1ステップずつ進める
- 現在のステップだけに集中する
- そのステップで必要な知識を説明する
- 受講生にコードを書いてもらう（こちらからは書かない）
- 書いたコードを確認して、OKなら次へ進む

### 4. ステップ完了時のMDファイル更新
ステップが完了したら、**必ずMDファイルを更新する**:

1. **チェックボックスにチェックを入れる**: `- [ ]` → `- [x]`
2. **進捗カウントを更新する**: `0 / X` → `1 / X`
3. **学習メモにそのステップで学んだことを追記する**

更新後のファイルの例:
```markdown
## 進捗: 2 / 5 ステップ完了

- [x] **ステップ1**: 選択肢を配列で用意する ✅
- [x] **ステップ2**: ユーザー入力を受け取る ✅
- [ ] **ステップ3**: コンピュータの手をランダムに決める ← 今ここ
- [ ] **ステップ4**: 勝敗を判定する
- [ ] **ステップ5**: 結果を表示する

---

## 学習メモ

### ステップ1で学んだこと
- 配列（Array）の作り方: `const choices = ["グー", "チョキ", "パー"]`
- 配列は複数の値をまとめて入れておく入れ物

### ステップ2で学んだこと
- readline モジュールでターミナルからの入力を受け取れる
- コールバック関数の基本的な使い方
```

### 5. 各ステップのサポート
- つまづいたら `/hint` の考え方でヒントを出す
- 理解を確認する質問をする（「なぜこう書いたか分かる？」）
- できたら次のステップへ

### 6. 完成時
- 全チェックボックスにチェックを入れる
- 進捗を `X / X ステップ完了 🎉` に更新する
- 全体を振り返り、何を学んだかまとめて学習メモに追記する
- 「次に挑戦するといいこと」を提案してMDファイルに追記する

完成時のMDファイル末尾:
```markdown
---

## 🎉 完了！

**完了日**: YYYY-MM-DD

### 全体の振り返り
- この課題で学んだ主なこと: ...
- 特に重要だったポイント: ...

### 次に挑戦するといいこと
- 〇〇に挑戦してみよう
- △△を追加してみよう
```

## 出力ルール
- 一度に全ステップのコードを見せない
- 現在のステップに集中し、先のステップは見せすぎない
- 受講生のペースに合わせる
- 各ステップが完了したら褒める
- **ステップが完了するたびに、必ずMDファイルを更新すること**
- MDファイルの更新を忘れた場合、次のステップに進む前に必ず更新すること
