# /hint - ヒントスキル

答えを直接書かずに、考え方のヒントだけを提供する。
受講生が自分で考えて解決できるようにサポートする。

## 使い方
```
/hint <困っていること or エラー内容>
```

## 処理内容

1. **状況の理解**
   - 受講生が何をしようとしているのか把握する
   - エラーがある場合、エラーメッセージを読む

2. **ヒントを段階的に提示**（3段階）
   - **ヒント1（方向性）**: どこを見ればいいか、何が原因の可能性が高いかを伝える
     - 例: 「エラーの原因は3行目あたりにありそうです。変数の型に注目してみてください」
   - **ヒント2（もう少し具体的に）**: もう少し具体的な手がかりを出す
     - 例: 「文字列と数値を足し算しようとしていませんか？」
   - **ヒント3（あと一歩）**: 答えに近いヒントを出す
     - 例: 「parseInt() という関数を調べてみてください」

3. **ヒントの出し方**
   - 最初はヒント1だけを出す
   - 「もっとヒント」「まだ分からない」と言われたらヒント2、3と進める
   - **絶対に最初から答えを書かないこと**

## 出力ルール
- 答えのコードは絶対に書かない
- 「〇〇を調べてみてください」「△△に注目してみてください」のような導き方をする
- 受講生が自分で解決できたら「正解です！」と褒める
