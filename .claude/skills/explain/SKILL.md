# /explain - コード解説スキル

指定されたコードを、初心者にも分かるように1行ずつ丁寧に解説する。
解説の根拠として、使用技術の**公式ドキュメント**を参照し、URLを添えて説明する。

## 使い方
```
/explain <ファイル名 or コードの場所>
```

## 処理内容

1. **指定されたコードを読み込む**
   - ファイル名が指定された場合、そのファイルを読み込む
   - 「この関数」「このファイル」など曖昧な指定でも、文脈から判断する

2. **使用技術の特定**
   - コードで使われている言語・フレームワーク・ライブラリを特定する
   - 例: HTML, CSS, JavaScript, React, Ruby on Rails, Python など

3. **全体像の説明**
   - このコードが「何をするためのものか」を最初に1-2文で説明する
   - 例: 「このファイルは、ユーザーの入力を受け取ってバリデーションする処理です」

4. **1行ずつ解説（公式ドキュメントを根拠に）**
   - コードを表示しながら、各行が何をしているか日本語で説明する
   - **公式ドキュメントを根拠にして説明すること**
   - 書き方のフォーマット:
     ```
     [行番号] コード
     → この行は〇〇をしています。△△という意味です。
     📖 公式ドキュメント: [説明の該当ページURL]
     ```
   - すべての行にURLが必要なわけではない。重要な構文・メソッド・パターンにURLを添える

5. **公式ドキュメントで推奨されている書き方の紹介**
   - 解説したコードについて、公式が推奨している書き方やベストプラクティスがあれば紹介する
   - 現在のコードが推奨と異なる場合は、推奨される書き方も見せて比較する
   - フォーマット:
     ```
     📖 公式の推奨:
     公式ドキュメントでは〇〇の書き方が推奨されています。
     URL: [公式ドキュメントのURL]

     推奨される書き方:
     （コード例）

     理由: △△のため
     ```

6. **重要ポイントのまとめ**
   - 特に覚えてほしい書き方やパターンをまとめる
   - 「初心者がつまづきやすいポイント」があれば補足する
   - 参照した公式ドキュメントのURLをまとめてリストにする

## 公式ドキュメントの参照ルール

- WebSearch ツールを使って公式ドキュメントの正確なURLを取得すること
- **推測でURLを作らないこと**。存在を確認できないURLは載せない
- 主要な技術の公式ドキュメント:
  - HTML/CSS: developer.mozilla.org (MDN)
  - JavaScript: developer.mozilla.org (MDN)
  - React: react.dev
  - Next.js: nextjs.org/docs
  - Ruby: docs.ruby-lang.org
  - Ruby on Rails: guides.rubyonrails.org / api.rubyonrails.org
  - Python: docs.python.org
  - Node.js: nodejs.org/docs
  - TypeScript: typescriptlang.org/docs
- 日本語版があれば日本語版のURLを優先する

## 出力ルール
- 専門用語には必ず（）で簡単な補足をつける
- 例え話を使って分かりやすくする
- 「なぜこの書き方をするのか」の理由も、公式ドキュメントの根拠とともに説明する
- 最後に「参照した公式ドキュメント」のリストをまとめること
