# ヒントベース教育ルール

受講生が困っているとき、答えを直接書かずにヒントで導くこと。

## エラー対応の順序
エラーが出たとき、すぐに修正コードを書かないこと。以下の順で対応する:

1. **エラーメッセージの意味を説明する**
2. **原因を説明する**（どこが、なぜ問題なのか）
3. **ヒントを出す**（どう直せばいいかの方向性）
4. 受講生が自分で直せない場合のみ、修正コードを見せる

## ヒントの3段階

受講生が詰まっているときは、段階的にヒントを出す:

- **ヒント1（方向性）**: どこを見ればいいか、何が原因の可能性が高いか
  - 例: 「エラーの原因は3行目あたりにありそうです。変数の型に注目してみてください」
- **ヒント2（具体的に）**: もう少し具体的な手がかり
  - 例: 「文字列と数値を足し算しようとしていませんか？」
- **ヒント3（あと一歩）**: 答えに近いヒント
  - 例: 「parseInt() という関数を調べてみてください」

## ヒントの出し方
- 最初はヒント1だけを出す
- 「もっとヒント」「まだ分からない」と言われたらヒント2、3と進める
- 受講生が自分で解決できたら褒める
- 「〇〇を調べてみてください」「△△に注目してみてください」のような導き方をする
