# 進捗管理ルール

複数ステップにわたる作業は、必ずMDファイルで進捗を管理すること。

## いつ適用するか
- 受講生が課題やタスクに取り組むとき
- 複数のステップが必要な作業のとき
- 「〇〇を作りたい」「〇〇に挑戦したい」と言われたとき

## 進捗MDファイルの作成

作業開始時に `progress_<課題名>.md` を作成する。

### ファイルのフォーマット:

```markdown
# 📋 <課題名> - 進捗タスクリスト

**ゴール**: <最終的にどうなればOKか>
**開始日**: YYYY-MM-DD

---

## 進捗: 0 / X ステップ完了

- [ ] **ステップ1**: 〇〇の説明
- [ ] **ステップ2**: △△の説明
- [ ] **ステップ3**: □□の説明
...

---

## 学習メモ
（ステップ完了時に学んだことを追記していく）
```

## ステップの分解
- ゴールを5-8個の小さなステップに分ける
- 1ステップずつ進め、現在のステップだけに集中する
- 先のステップのコードは見せない

## ステップ完了時の更新
ステップが完了したら、**必ず**MDファイルを更新する:

1. チェックボックスにチェック: `- [ ]` → `- [x]`
2. 進捗カウントを更新: `0 / X` → `1 / X`
3. 学習メモにそのステップで学んだことを追記

### 更新後の例:
```markdown
## 進捗: 2 / 5 ステップ完了

- [x] **ステップ1**: 選択肢を配列で用意する ✅
- [x] **ステップ2**: ユーザー入力を受け取る ✅
- [ ] **ステップ3**: コンピュータの手を決める ← 今ここ
- [ ] **ステップ4**: 勝敗を判定する
- [ ] **ステップ5**: 結果を表示する

---

## 学習メモ

### ステップ1で学んだこと
- 配列の作り方: `const choices = ["グー", "チョキ", "パー"]`
```

## 完了時
- 全チェックボックスにチェックを入れる
- 全体の振り返り・学んだことのまとめを追記する
- 「次に挑戦するといいこと」を提案して追記する

```markdown
## 🎉 完了！
**完了日**: YYYY-MM-DD

### 全体の振り返り
- この課題で学んだ主なこと: ...

### 次に挑戦するといいこと
- 〇〇に挑戦してみよう
```
